How it works.

Analog Voltage Output

The TMP36 is a linear analog temperature sensor. Unlike digital sensors that send complex data packets, the TMP36 outputs a simple voltage on its middle pin that is directly proportional to the temperature.

The scale is 10 millivolts per degree Celsius. To handle negative temperatures without negative voltage, the sensor adds a 500mV offset. This means at 0¬∞C, the output is 0.5V.

Move the slider in the demo above to see how the output voltage changes in real-time with temperature.

Simulation note (Tinkercad): The TMP36 component in Tinkercad is a simplified simulation and does not always output the exact ADC values a physical TMP36 would. The demo sketch on this page uses a mapping tuned to Tinkercad's simulator so the displayed temperature appears correct inside the simulator. This mapping is not the actual TMP36 equation. For real hardware use: Temp¬∞C = (Vout ‚àí 0.5) √ó 100.

Inside the system.

Linear Output

Linear slope of 10mV/¬∞C makes calculation incredibly simple without complex lookup tables.

Instant Response

Because it is purely analog, there is zero communication latency. The voltage changes instantly as temperature shifts, limited only by the Arduino's ADC speed.

No Libraries Required

Unlike DHT11 or DS18B20, you don't need to install any external libraries. Just use `analogRead()` and simple math.

Low Power

Draws less than 50 ¬µA of current, making it excellent for battery-powered loggers.

Analog Precision

10-bit ADC resolution means ~0.5¬∞C precision at room temperature.

Simple to code.

// TINKERCAD ONLY ‚Äî simulator mapping + Adafruit I2C LCD
#include <Adafruit_LiquidCrystal.h>

Adafruit_LiquidCrystal lcd_1(0); // 0 = select module in Tinkercad UI

void setup() {
  pinMode(A0, INPUT);
  Serial.begin(9600);

  lcd_1.begin(16, 2);
  lcd_1.backlight();     // ensure backlight is on in Tinkercad
}

void loop() {
  int raw = analogRead(A0);

  // Tinkercad-tuned mapping (simulator only)
  // Keep as integer math because Tinkercad's TMP36 is simulated.
  int temp = map((int)((raw - 20) * 3.04), 0, 1023, -20, 120);

  // Debug - helpful if LCD stays blank
  Serial.print("raw=");
  Serial.print(raw);
  Serial.print("  temp=");
  Serial.println(temp);

  lcd_1.clear();                 // prevents ghosting from previous text
  lcd_1.setCursor(0, 0);
  lcd_1.print("Temp Monitr Sys");
  lcd_1.setCursor(0, 1);
  lcd_1.print("Temp=");
  lcd_1.setCursor(6, 1);
  lcd_1.print(temp);
  lcd_1.print(" C    ");          // trailing spaces clear leftover chars

  delay(500);                    // stable, readable update rate
}

Alternative: Real Hardware Code

// REAL HARDWARE ‚Äî actual TMP36 conversion
#include <LiquidCrystal_I2C.h>
LiquidCrystal_I2C lcd(0x20, 16, 2); // change address to your module (0x27 or 0x3F, etc)

void setup() {
  Serial.begin(9600);
  lcd.init();
  lcd.backlight();
}

void loop() {
  int reading = analogRead(A0);
  float voltage = reading * (5.0 / 1024.0);  // use 3.3 if your ref is 3.3V
  float tempC = (voltage - 0.5) * 100.0;

  lcd.setCursor(0,0);
  lcd.print("Temp: ");
  lcd.print(tempC, 1);
  lcd.print(" C   ");

  lcd.setCursor(0,1);
  lcd.print("Volt: ");
  lcd.print(voltage, 2);
  lcd.print(" V   ");

  delay(500);
}

Common Mistakes

Reading > 100¬∞C or fluctuates wildly

Check Ground: If the GND pin disconnects, the reading floats. Also, ensure you are using 5.0V in your math if your board is 5V.

Stabilize Power: Add a 0.1¬µF capacitor between VCC and GND close to the sensor to filter noise.

Sensor gets hot

Reverse Polarity: You likely wired 5V and GND backwards. Unplug immediately to save the sensor!

Over-voltage: TMP36 max voltage is 5.5V. Applying higher voltage will damage it.
